# Product Demo

Use when the footage shows a product, feature, workflow, or product-led explanation.

Common structure:
PROBLEM -> FEATURE -> DEMO -> BENEFIT -> CTA

Prioritize:
- problem the feature solves;
- visible demo or concrete explanation;
- benefit stated in user terms;
- proof that the feature works;
- clean CTA or next step.

Cut:
- setup that delays seeing the product;
- repeated feature descriptions;
- UI waiting time;
- vague benefit claims;
- tangents unrelated to the demo.

Common mistakes:
- keeping explanation without enough demo;
- showing feature details without the problem;
- cutting away before the benefit lands.

Target runtime guidance:
- fit the brief; otherwise keep enough product context for the demo to make sense.

Do not force all beats if the material does not support them.
