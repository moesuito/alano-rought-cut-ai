# Social Talking-head

Use when one speaker delivers direct-to-camera social content, thought leadership, a personal point, or a compact opinion.

Common structure:
HOOK -> CONTEXT -> MAIN POINTS -> PAYOFF -> CTA / END BEAT

Prioritize:
- strongest first sentence or surprising claim;
- clear thesis;
- concise supporting points;
- punchline, payoff, or memorable framing;
- clean CTA or natural end beat.

Cut:
- repeated intros;
- throat clearing before the hook;
- redundant examples;
- long context that delays the point;
- weaker duplicate versions of the same claim.

Common mistakes:
- over-tightening until delivery feels robotic;
- keeping source order when a later hook belongs first;
- cutting pauses that create emphasis.

Target runtime guidance:
- usually 30-90s, but let the material decide.

Do not force all beats if the material does not support them.
