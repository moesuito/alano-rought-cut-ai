# Testimonial / Case Study

Use when the speaker describes a before/after experience, result, transformation, client story, product/service experience, or credibility proof.

Common structure:
HOOK / RESULT -> WHO THEY ARE -> PROBLEM BEFORE -> SOLUTION / CHANGE -> RESULT -> CLOSING STATEMENT

Prioritize:
- concrete numbers/results;
- strong pain/problem statements;
- clear before/after contrast;
- credibility details;
- emotionally believable delivery;
- concise closing statement.

Cut:
- repeated introductions;
- vague praise without proof;
- long context that delays the result;
- weaker duplicate versions of the same claim.

Common mistakes:
- opening with biography instead of outcome;
- keeping too much setup;
- forcing a CTA when a closing statement is stronger.

Target runtime guidance:
- usually 60-120s.

Do not force all beats if the material does not support them.
